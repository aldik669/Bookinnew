# Календарь МК (для менеджеров)

Внутренний сайт-календарь, который показывает расписание мастер-классов и загруженность
каждого слота на основе сделок из amoCRM (этап «Назначено пробное»).

## Установка

```bash
npm install
```

## Настройка

1. Скопируйте `.env.example` в `.env` (если файла `.env` ещё нет — он уже создан).
2. Заполните `AMO_SUBDOMAIN` — поддомен вашего amoCRM (например, для
   `https://mycompany.amocrm.ru` это `mycompany`).
3. `AMO_TOKEN` — долгосрочный Bearer-токен интеграции amoCRM.
4. Запустите утилиту, чтобы найти ID воронки, этапа и поля:

   ```bash
   npm run ids
   ```

   Она выведет в консоль список воронок с этапами и список полей сделок, в названии
   которых встречаются «мк», «дата» или «время» — найдите среди них:
   - воронку, в которой ведутся МК → впишите её ID в `AMO_PIPELINE_ID`;
   - этап «Назначено пробное» в этой воронке → `AMO_STATUS_ID`;
   - поле «Дата и Время МК (свое)» → `AMO_FIELD_MK_ID`.

5. При необходимости поменяйте `SLOT_LIMIT` (по умолчанию 16) и `PORT` (по умолчанию 3000).

## Запуск

```bash
npm start
```

Откройте [http://localhost:3000](http://localhost:3000).

## Как это работает

- `amo.js` — ходит в amoCRM API v4, постранично выгружает сделки этапа «Назначено пробное»,
  парсит кастомное поле «Дата и Время МК (свое)» (timestamp или строка), группирует сделки
  по дате+времени в слоты и подтягивает имена/телефоны через `/api/v4/contacts`.
- `server.js` — Express-сервер, кеширует результат `amo.js` на 30 секунд и отдаёт его на
  `GET /api/slots`. Токен и ID никогда не уходят на фронт.
- `public/` — чистый HTML/CSS/JS без фреймворков. Раз в 30 секунд опрашивает `/api/slots`
  и перерисовывает календарь.

## Безопасность

`AMO_TOKEN` хранится только в `.env` на сервере и в `.gitignore`. Браузер обращается
только к собственному backend'у (`/api/slots`), токен ему не передаётся.

## Если что-то не работает

- Статус-индикатор в правом верхнем углу страницы показывает «ошибка связи» и текст
  ошибки amoCRM (например, протухший токен или неверные ID) — сервер при этом не падает.
- Проверьте, что все 5 переменных в `.env` заполнены (`npm run ids` поможет найти ID).
